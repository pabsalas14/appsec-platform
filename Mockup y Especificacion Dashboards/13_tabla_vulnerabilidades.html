SQL Injection (CWE-89)

El software construye toda o parte de una sentencia SQL utilizando entrada de componentes externos, pero no neutraliza o neutraliza incorrectamente los elementos especiales que podrían modificar la sentencia SQL prevista.

Vulnerabilidad Confirmada

El snippet de código muestra una concatenación directa de la variable req.getParameter("user") dentro de la consulta SQL en la línea 42. Al no utilizar PreparedStatement ni sanitizar la entrada, el código es altamente vulnerable a inyección SQL. No se detectan controles compensatorios en el framework Spring Boot para este patrón específico de uso de Statement nativo.
